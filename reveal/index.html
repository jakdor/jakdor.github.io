### Synchroniczność vs Asynchroniczność ![Synch vs async](img/synchronous-asynchronous-javascript.png) -- [phpmind.com](http://www.phpmind.com/blog/2017/05/synchronous-and-asynchronous/)
### Wzorzec obserwator - Zależność 1-* między obiektami bez ścisłego powiązania - Zmiana stanu/wartości w jednym obiekcie powoduje zmianę w dowolnej ilości innych obiektów - Jeden obiekt ma możliwość powiadomienia dowolnej ilości innych obiektów
![Observer pattern example](img/Observer_example1.png) -- [sourcemaking.com/design_patterns/observer](https://sourcemaking.com/design_patterns/observer)
![Observer pattern schema](img/httpatomoreillycomsourceoreillyimages1547801.png) -- [sourcemaking.com/design_patterns/observer](https://sourcemaking.com/design_patterns/observer)
| | pojedyńczy obiekt | wiele obiektów | | -------------- |:--------------------:| ---------------------:| | Synchronicznie | `T getData()` | `Iterable<T> getData()` | | Asynchronicznie| `Future<T> getData()`| `Observable<T> getData()`|
| | Iterable (pobrane) | Observable (odebrane) | | -------------- |:--------------------:| ---------------------:| | Pobieranie danych | `T next()` | `onNext(T)` | | Wyjątki | `throw Exception`| `onError(Exception) `| | Ostatni element| `!hasNext()`| `onCompleted()`|
